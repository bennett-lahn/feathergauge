## Feather 32u4 Multi-Board Programming Guide

This guide provides information on setup, configuration, usage, two‑stage programming, examples, and troubleshooting for `program_feather_boards.sh`.

### What the script does
- **Checks tools**: arduino-cli, avrdude, python3
- **Sets up Arduino CLI** and required libraries from `library_config/`
- **Compiles** the main sketch (and RTC setup sketch)
- **Detects** connected Feather 32u4 boards
- **Programs** each board (two‑stage programming, RTC and main script, by default)

## Prerequisites
- Linux/Unix environment (script targets Linux/macOS shells)
- Installed: `arduino-cli`, `avrdude`, `python3`
- `library_config/` folder with required Arduino libraries

Quick install examples:
```bash
# Ubuntu/Debian
curl -fsSL https://raw.githubusercontent.com/arduino/arduino-cli/master/install.sh | sh \
  && sudo mv bin/arduino-cli /usr/local/bin/
sudo apt-get update && sudo apt-get install -y avrdude python3

# macOS (Homebrew)
brew install arduino-cli avrdude python3
```

## Library Setup
The script installs libraries from local files in `library_config/` instead of downloading them. This ensures consistent builds and version control.

### Required Libraries
- **RTClib** - Real-time clock library
- **TimerOne** - Timer interrupt library  
- **LowPower** - Power management library
- **SparkFun MS5803** - Pressure sensor library
- **MS5837** - Alternative pressure sensor library

### Library File Formats
- Compressed archives: `.zip`, `.tar.gz`, `.tar.bz2`, `.tar.xz`
- Library directories: containing `library.properties` or `library.json`

### Quick Library Setup
```bash
# Create library directory
mkdir -p library_config

# Download required libraries (example)
cd library_config
wget https://github.com/adafruit/RTClib/archive/refs/heads/master.zip -O RTClib.zip
wget https://github.com/PaulStoffregen/TimerOne/archive/refs/heads/master.zip -O TimerOne.zip
wget https://github.com/LowPowerLab/LowPower/archive/refs/heads/master.zip -O LowPower.zip
wget https://github.com/sparkfun/SparkFun_MS5803-14BA_Breakout_Arduino_Library/archive/refs/heads/main.zip -O SparkFun_MS5803.zip
wget https://github.com/bluerobotics/MS5837/archive/refs/heads/master.zip -O MS5837.zip
```

## Configuration (optional)
Copy `programming_config.example` to `programming_config` and adjust as needed.

Key fields typically customized:
- **Board**: `BOARD_FQBN="adafruit:avr:feather32u4"`
- **Sketch**: `SKETCH_DIR`, `SKETCH_NAME`
- **Build flags**: `BUILD_FLAGS` (e.g. `-DUSE_NEW_SENSOR=false -DDELAY_START=false -DBURST_SAMPLING=false`)
- **Programmer/baud**: `PROGRAMMER`, `BAUD_RATE`
- **Device patterns**: `DEVICE_PATTERNS` (e.g. `/dev/ttyUSB* /dev/ttyACM* ...`)
- **Logging/output**: `COMPILED_SKETCH_DIR`, `LOG_DIR`, `LOG_LEVEL`

## Quick start
```bash
chmod +x program_feather_boards.sh
./program_feather_boards.sh
```
Expected output includes dependency checks, compilation, found devices, and success results per board.

### Common options
```bash
./program_feather_boards.sh [OPTIONS]
  -h, --help               Show help
  -s, --sketch DIR         Sketch directory (defaults to config)
  -o, --output DIR         Output directory for compiled sketches
  -l, --log FILE           Log file path
  --skip-rtc               Program main sketch only (skip two‑stage)
```

## Two‑stage programming
Ensures all boards start with accurate RTC time.

1) **Stage 1 (RTC setup)**
- Compiles and flashes `rtc_setup.ino`
- Sends current system time over serial and waits for success
2) **Stage 2 (Main program)**
- Compiles and flashes the main sketch

Defaults used by the serial helper:
- Baud: 57600
- Timeout: ~30s
- Time format: `YYYY-MM-DD HH:MM:SS`

Run with two‑stage (default):
```bash
./program_feather_boards.sh
```
Skip RTC setup if not needed:
```bash
./program_feather_boards.sh --skip-rtc
```

## Examples
Basic:
```bash
./program_feather_boards.sh                       # default flow
./program_feather_boards.sh -s ./my_sketch        # custom sketch dir
./program_feather_boards.sh -o ./build_output     # custom compile output
./program_feather_boards.sh -l programming.log    # custom log file
```

Batch multiple sketches:
```bash
#!/usr/bin/env bash
for sketch in \
  jorch_featherDIY_customizable \
  jorch_featherDIY_freertos \
  featherDIYv4_rapidstart_continuous_v1; do
  ./program_feather_boards.sh -s "Sketchbooks/Codes_v1/$sketch"
  sleep 5
done
```

Debug mode:
```bash
ARDUINO_CLI_DEBUG=1 ./program_feather_boards.sh
```

## Troubleshooting
- **No devices detected**: Check USB cable/ports; ensure proper permissions (Linux):
  ```bash
  sudo usermod -a -G dialout "$USER"  # then log out/in
  ```
- **Permission denied**: See dialout group step above; avoid ports in use
- **Compilation failed**: Verify libraries in `library_config/` and sketch paths
- **RTC setup failed**: Confirm RTC hardware, baud 57600, and time format; ensure `pyserial` is installed (`pip3 install pyserial` or `sudo apt-get install python3-serial`)
- **Programming errors**: Replug board; try single board; confirm correct board FQBN and programmer

### Library Issues
- **"No library files found in library_config/"**: Ensure directory exists and contains library files
- **"Failed to install: [library_name]"**: Check file integrity and Arduino library structure
- **"Library not found during compilation"**: Verify installation with `arduino-cli lib list`
- **Library management**:
  ```bash
  arduino-cli lib list                    # List installed libraries
  arduino-cli lib uninstall [name]       # Remove library
  arduino-cli lib install library_config/[name].zip  # Install from local file
  ```

### Logs
Script emits timestamped logs like `programming_log_YYYYMMDD_HHMMSS.log` containing output, errors, and per‑board status.
